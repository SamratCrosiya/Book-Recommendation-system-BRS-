BOOK RECOMMENDATION SYSTEM - COMPLETE CODEBASE EXPLANATION
===========================================================

SHORT PROJECT DESCRIPTION
=========================

BookLens is a Flask-based book recommendation web app that helps users discover similar books from a selected title. It uses collaborative filtering and cosine similarity on user rating data to find books with similar reading patterns. The processed model data is stored in pickle files, making the website fast because it does not need to reprocess large CSV datasets every time it runs. The app includes a home page for top popular books and a recommendation page where users can search for a book and receive similar suggestions.


Project name:
BookLens / Book Recommendation System

Main idea:
This project is a Flask web application that shows popular books and recommends similar books based on user rating patterns.

In simple words:
When a user enters a book title, the system checks which other books have similar rating behavior from users and returns the closest matching books.


1. SHORT INTRODUCTION TO SAY TO TEACHER
=======================================

This is a book recommendation web application built using Python Flask.

The project uses a collaborative filtering recommendation approach. It does not recommend books by reading their story, genre, or author. Instead, it uses user rating behavior. If many users rated two books in a similar way, then those books are considered similar.

The heavy data processing was done earlier in the Jupyter notebook. The final processed data was saved into pickle files. The Flask app loads those pickle files directly, so the website can start faster and does not need to process the large CSV files every time.

The project has two main pages:

1. Home page:
   Shows the top 50 popular books.

2. Recommend page:
   Takes a book title from the user and shows 4 similar books.

7. DATA PROCESSING FLOW IN NOTEBOOK
===================================

The notebook prepares the recommendation data.

Step 1:
Load Books.csv and Ratings.csv.

Step 2:
Merge ratings with book names using ISBN.

Why?
Ratings.csv has ISBN, but for display and recommendation we need book titles.

Step 3:
Create popular_df.

For every book:

- Count how many ratings it has
- Calculate average rating
- Keep only books with at least 250 ratings
- Sort by average rating
- Take top 50 books

This becomes popular.pkl.

Step 4:
Filter active users.

Only users who rated more than 200 books are selected.

Why?
Users with very few ratings may not give enough useful information.

Step 5:
Filter famous books.

Only books that received at least 50 ratings from active users are selected.

Why?
Books with very few ratings may produce weak or noisy recommendations.

Step 6:
Create pivot table.

Rows:
Book titles

Columns:
User IDs

Values:
Ratings

Missing ratings:
Filled with 0

This becomes pt.pkl.

Step 7:
Calculate cosine similarity.

Each book is treated like a vector of user ratings.
Cosine similarity compares the direction of two rating vectors.

If two books were rated similarly by users, their cosine similarity is high.

This becomes similarity_scores.pkl.

Step 8:
Save required book details.

Book title, author, and image URL are saved into books.pkl.


8. WHAT IS COLLABORATIVE FILTERING?
===================================

Collaborative filtering recommends items based on user behavior.

In this project:

- Users give ratings to books.
- The system compares books based on rating patterns.
- If the same type of users rated two books similarly, the books are considered similar.

Example:

If users who liked "Harry Potter" also liked "The Hobbit", then the system may recommend "The Hobbit" when someone enters "Harry Potter".

This is called item-based collaborative filtering because the system compares items, meaning books.


9. WHAT IS COSINE SIMILARITY?
=============================

Cosine similarity is a method to measure how similar two vectors are.

In this project:

Each book becomes a vector of user ratings.

Example:

Book A ratings:
[5, 0, 8, 10]

Book B ratings:
[4, 0, 9, 10]

These two books have a similar pattern, so their cosine similarity will be high.

Cosine similarity value is usually between 0 and 1:

0 means not similar.
1 means very similar.


10. WHAT IS PICKLE?
===================

Pickle is a Python module used to save Python objects into files.

For example, a pandas DataFrame can be saved as:

popular.pkl

Later, the Flask app can load it back using:

pickle.load(...)

Why pickle is used here:

The original CSV files are large.
If the app processed them every time a user opened the website, it would be slow.

So the notebook processes the data once and saves the final objects as pickle files.
Then the Flask app directly loads those ready-made objects.

This makes the app faster and simpler at runtime.

Important point:
Pickle files should only be loaded if they are created by us or from a trusted source.
Loading unknown pickle files can be unsafe because pickle can execute Python object loading logic.

12. HOW RATINGS ARE SHORTENED
=============================

Function:

def format_rating(value):
    return f'{np.floor(float(value) * 100) / 100:.2f}'

Purpose:
This changes long rating values like:

5.737410071942446

into:

5.73

Step by step:

float(value)
Converts the value into a number.

float(value) * 100
Moves the decimal two places forward.

5.737410071942446 becomes 573.7410071942446

np.floor(...)
Cuts off the extra decimal part.

573.7410071942446 becomes 573

/ 100
Moves the decimal back.

573 becomes 5.73

:.2f
Always shows two digits after the decimal.

Example:

5.7 becomes 5.70

This is applied to all ratings using a list comprehension:

[format_rating(value) for value in popular_df['avg_rating'].values]


13. RECOMMENDATION FUNCTION EXPLANATION
=======================================

Route:

@app.route('/recommend_books', methods=['POST'])
def recommend():

Meaning:
This route handles the submitted recommendation form.


Step 1:
Read user input.

user_input = request.form.get('user_input', '').strip()

Meaning:
Get the book name typed by the user.
strip() removes extra spaces from the beginning and end.


Step 2:
Set book_title.

book_title = user_input

Meaning:
By default, the book title is same as user input.


Step 3:
Case-insensitive matching.

if user_input and user_input not in pt.index:
    matches = [title for title in pt.index if title.lower() == user_input.lower()]
    if matches:
        book_title = matches[0]

Meaning:
If the exact title is not found, the app tries to match it by ignoring uppercase and lowercase differences.

Example:

If dataset has:
"1984"

and user types:
"1984"

It matches directly.

If dataset has:
"The Notebook"

and user types:
"the notebook"

The lowercase comparison can still find it.


Step 4:
Handle missing book.

if not book_title or book_title not in pt.index:
    return render_template('recommend.html', data=[], error="Book not found. Please try another title.")

Meaning:
If the book is empty or not present in the pivot table, show an error message.


Step 5:
Find selected book index.

index = np.where(pt.index == book_title)[0][0]

Meaning:
pt.index contains all book titles used in the recommendation model.
This line finds the position number of the selected book.


Step 6:
Get similar items.

similar_items = sorted(
    list(enumerate(similarity_scores[index])),
    key=lambda x: x[1],
    reverse=True
)[1:5]

Meaning:

similarity_scores[index]
Gets similarity scores for the selected book against all 706 books.

enumerate(...)
Adds index numbers to each similarity score.

sorted(...)
Sorts books by similarity score.

reverse=True
Highest similarity first.

[1:5]
Skips the first item and selects the next 4 books.

Why skip the first item?
The most similar book to a book is the same book itself.
Its similarity score is usually 1.
So we skip it and show the next 4 books.


Step 7:
Create result data.

data = []
for i in similar_items:
    temp_df = books[books['Book-Title'] == pt.index[i[0]]]
    temp_df = temp_df.drop_duplicates('Book-Title')
    item = [
        temp_df['Book-Title'].values[0],
        temp_df['Book-Author'].values[0],
        temp_df['Image-URL-M'].values[0],
    ]
    data.append(item)

Meaning:
For each similar book, get:

- Title
- Author
- Image

Then add it to data.


Step 8:
Send result to HTML.

return render_template('recommend.html', data=data)

Meaning:
The recommendation page receives data and displays the recommended book cards.

20. LIMITATIONS OF THIS PROJECT
===============================

1. Book title must exist in pt.pkl.
If the user enters a book not present in the pivot table, the app cannot recommend.

2. It uses old rating data.
Recommendations depend on the dataset, not live data.

3. It does not understand book content.
It does not analyze genre, plot, language, or description.

4. It recommends only 4 books.
The code takes [1:5], which gives 4 recommendations.

5. Pickle files must match the installed pandas/numpy versions.
If pickles were created with very old versions, loading them in a newer version may fail.

2. Web application:
Done in Flask.
It loads pickle files and serves pages to the user.

The recommendation logic is:

Input book title
        |
Find book index in pt.pkl
        |
Read similarity scores from similarity_scores.pkl
        |
Sort books by similarity
        |
Skip the same book
        |
Get title, author, image from books.pkl
        |
Show recommendations in recommend.html

This is how the complete system works from data to final output.


25. HOW TO START THE FRONTEND / WEBSITE
=======================================

To start the website, open PowerShell or terminal in the project root folder.

Project root means this folder:

Book Recommendation/

Then run:

py frontend\app.py

After running the command, Flask will start the local development server.

Then open this URL in the browser:

http://127.0.0.1:5000/

This opens the home page of the BookLens project.

To open the recommendation page directly, use:

http://127.0.0.1:5000/recommend


If port 5000 is already busy
----------------------------

Run the app on another port, for example 5001:

$env:PORT="5001"; py frontend\app.py

Then open:

http://127.0.0.1:5001/


Important note
--------------

On this machine, the python command may not work because it points to the Windows Store shortcut.
So use:

py frontend\app.py

instead of:

python frontend\app.py


How to stop the server
----------------------

Go back to the terminal where Flask is running and press:

Ctrl + C

This stops the local Flask server.
